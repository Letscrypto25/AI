# Pattern Predictor

A robust, intelligent **pattern-based time series forecasting tool** built as a zero-dependency vanilla web app. It identifies the underlying structure of a data series — cyclic, trending, seasonal, stationary, or noisy — then generates predictions using a multi-perspective ensemble approach.

![Pattern Predictor screenshot](https://via.placeholder.com/900x500/0b0c0f/e8c547?text=Pattern+Predictor)

---

## Features

- **Pattern identification** — automatically classifies series as cyclic, trend, seasonal, stationary, or noisy
- **Period detection** — autocorrelation (ACF) analysis finds the dominant repeating lag
- **4 independent forecasting perspectives** — Cyclic, Trend, Moving Average, Constraint-based
- **Ensemble weighting** — weights shift dynamically based on pattern type, cycle count, and signal consistency
- **Confidence bands** — uncertainty grows with prediction horizon, scaled by signal variability
- **Diagnostic feedback loop** — records prediction errors for continuous recalibration
- **Interactive canvas** — click to plot points, right-click to remove; or load a preset
- **5 built-in presets** — Sine, Trend, Seasonal, Noisy, Stationary

## Usage

### Plotting data

| Action | Result |
|---|---|
| **Left-click** on canvas | Add a data point |
| **Right-click** on canvas | Remove the last point |
| **Preset buttons** | Load a sample pattern (Sine, Trend, Seasonal, Noisy, Stationary) |

### Running a forecast

1. Plot at least **4 data points** on the canvas (or load a preset)
2. Adjust **Prediction Points** (1–10) to set how far ahead to forecast
3. Adjust **Noise Level** to simulate signal noise when using presets
4. Click **Run Forecast ◈**

### Reading the output

The canvas renders five layers simultaneously:

| Color | Perspective |
|---|---|
| 🟢 Teal | Cyclic forecast — phase-aligned from historical positions |
| 🟠 Orange | Trend forecast — linear regression extrapolation |
| 🟣 Purple | Moving average — smoothed short-term projection |
| 🩷 Pink | Constraint-based — bound-aware regression to mean |
| 🟡 Yellow | **Ensemble** — weighted combination of all four (the main prediction) |

The **sidebar diagnostics** panel shows the detected pattern type, period, amplitude, trend slope, cycle count, confidence score, and the live ensemble weights for each perspective.

## How It Works

### 1. Pattern Identification

The engine computes a **linear slope** (least-squares regression) and **cyclic strength** (Pearson correlation between adjacent cycles) to classify the series:

| Type | Condition |
|---|---|
| `seasonal` | High trend strength **and** high cyclic strength |
| `cyclic` | High cyclic strength, low trend |
| `trend` | High trend strength, low cyclic |
| `stationary` | Low std relative to mean |
| `noisy` | None of the above |

### 2. Period Detection

The series is **detrended**, then an **autocorrelation function (ACF)** is computed up to `min(n/2, 30)` lags. The first significant local peak with `ACF > 0.2` is taken as the dominant period.

### 3. Ensemble Perspectives

Each perspective generates `n` predicted values independently:

- **Cyclic** — for each future index, finds all historical values at the same phase position (`index % period`), averages them, and re-applies the trend
- **Trend** — extends the least-squares regression line forward
- **Moving Average** — computes a window average of recent values and blends with slope
- **Constraint** — gently pulls predictions toward the historical midpoint, respecting observed min/max bounds

### 4. Dynamic Weighting

Weights are assigned based on pattern type and evidence quality:

```
cyclic weight  → scaled by cyclicStrength × min(cycles/2, 1)
trend weight   → scaled by trendStrength
movingAvg      → default high weight for noisy/stationary signals
constraint     → high weight when bounds are tight
```

All weights are normalized to sum to 1.

### 5. Constraint Clamping

Final ensemble predictions are soft-clamped to `[observed_min - 5% range, observed_max + 5% range]` to prevent unrealistic outliers.

### 6. Feedback Loop

```js
// After observing actual outcomes:
predictor.recordFeedback(predictedArray, actualArray);

// Check average prediction error:
const avgErr = predictor.averageError();
```

The predictor stores up to 50 historical error records. This data can be used externally to recalibrate noise estimates or re-weight perspectives over time.

## Integrating the Predictor Engine

`predictor.js` is fully self-contained and exports `window.PatternPredictor`. You can use it independently:

```html
<script src="predictor.js"></script>
<script>
  const predictor = new PatternPredictor();

  const series = [1.2, 1.8, 2.4, 1.9, 1.3, 1.7, 2.5, 2.0, 1.4];
  const result = predictor.predict(series, 3); // predict next 3 points

  console.log(result.predictions);   // [2.1, 1.8, 1.5] (example)
  console.log(result.diagnostics);   // { type, period, amplitude, slope, confidence, ... }
  console.log(result.weights);       // { cyclic: 0.4, trend: 0.1, movingAvg: 0.3, constraint: 0.2 }
  console.log(result.perspectives);  // individual forecasts per method
</script>
```

### `predictor.predict(series, n)`

| Parameter | Type | Description |
|---|---|---|
| `series` | `number[]` | Array of numeric values (minimum 4) |
| `n` | `number` | Number of future points to predict (default: 3) |

**Returns:**

```js
{
  predictions: number[],      // ensemble forecast (n values)
  diagnostics: {
    type: string,             // 'cyclic' | 'trend' | 'seasonal' | 'stationary' | 'noisy'
    period: number,           // detected period (0 if none)
    amplitude: number,        // peak-to-peak / 2
    slope: number,            // linear trend slope
    cycles: number,           // estimated cycles completed
    confidence: number,       // 0–1
    constraints: { lower, upper },
    mean: number,
    std: number
  },
  perspectives: {
    cyclic: number[],
    trend: number[],
    movingAvg: number[],
    constraint: number[]
  },
  weights: {
    cyclic: number,
    trend: number,
    movingAvg: number,
    constraint: number
  }
}
```
