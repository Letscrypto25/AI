# HEARTBEAT.md

## Routine

- Re-orient to the active repo at the start of a session
- Use a 15-minute heartbeat target for planning and tracking
- Check `Actions/requests.md` on each wake for new or unclaimed work
- Review the workspace control files when present, especially `brain.md`, `Actions/todo.md`, `CONSTRAINTS.md`, `PERMISSIONS.md`, `WORKFLOW.md`, `mind/MEMORY.md`, and `mind/thoughts.md`
- Follow the rules in `Actions/requests.md` before acting on any request
- When a request is actionable, claim it with a stable ID, then immediately use `Actions/todo.md` to carry out and track the work
- Keep `Actions/todo.md` accurate for this session's claimed work: mark it pending when claimed, complete when fully done, and leave it pending if work pauses mid-task
- Do not pause while actionable Saved Games requests remain open unless the work is blocked by the Saved Games boundary
- Record meaningful heartbeat work in `Actions/timeline.md`
- Prefer finishing one concrete coding task well over juggling many
- Keep the user updated with short progress notes when work takes time
